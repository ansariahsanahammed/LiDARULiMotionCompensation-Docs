\chapter{C\_py\_files\_gen.py}

This script downsamples a \texttt{.las} point cloud to 10\% and 1\% of the original dataset. The downsampled point clouds are saved as both \texttt{.las} and \texttt{.csv} files. The \texttt{.csv} files are used as input for the boresight calibration GUI (\autoref{ch:scriptE})

\section{How It Works}

The \texttt{.las} file is read using the \texttt{laspy} Python library. Downsampling is done by retaining every n\textsuperscript{th} point from the input file - every 10\textsuperscript{th} point for 10\% downsampling, and every 100\textsuperscript{th} point for 1\% downsampling. The retained points along with all their associated attributes are written to a new \texttt{.las} file using \texttt{laspy}.

A \texttt{.csv} file is then generated from each downsampled \texttt{.las} file using \texttt{numpy}. The \texttt{.csv} files contain only the fields \texttt{gps\_time, x, y, z}.

\section{Outputs}
All outputs are saved to the same directory as the input \texttt{.las} file.
\begin{itemize}
    \item \texttt{10\%\_<filename>.las}: 10\% downsampled point cloud
    \item \texttt{1\%\_<filename>.las}: 1\% downsampled point cloud
    \item \texttt{10\%\_<filename>.csv}: CSV with columns: \texttt{gps\_time, x, y, z}
    \item \texttt{1\%\_<filename>.csv}: CSV with columns: \texttt{gps\_time, x, y, z}
\end{itemize}
