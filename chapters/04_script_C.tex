\chapter{C\_py\_files\_gen.py}

This script downsamples a \texttt{.las} point cloud to 10\% and 1\% of the original dataset. The downsampled point clouds are then saved as both \texttt{.las} and \texttt{.csv} files. The \texttt{.csv} files are used as input for the boresight calibration GUI (\autoref{ch:scriptE})

\section{Process}

The \texttt{.las} files are read using the \texttt{laspy} Python library, and the downsampling is done by retaining every n\textsuperscript{th} (10\textsuperscript{th} and 100\textsuperscript{th} for 10\% and 1\% respectively) point from the input \texttt{.las} file. These retained points and its associated attributes are then saved as a new \texttt{.las} file.

A simple human-readable \texttt{.csv} file of an input \texttt{.las} file is generated which only contain the fields \texttt{gps\_time, x, y, z} for simpler tasks and analysis.

\section{Outputs}
All outputs are saved to the same directory as the input .las file:
\begin{itemize}
    \item \texttt{10\%\_<filename>.las}: 10\% downsampled point cloud
    \item \texttt{1\%\_<filename>.las}: 1\% downsampled point cloud
    \item \texttt{10\%\_<filename>.csv}: CSV with columns: \texttt{gps\_time, x, y, z}
    \item \texttt{1\%\_<filename>.csv}: CSV with columns: \texttt{gps\_time, x, y, z}
\end{itemize}
